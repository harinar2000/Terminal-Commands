Terminal & Git

Every command you actually need — from navigating a folder to pushing a branch. No filler.

Git flow → init / clone → branch → stage → commit → push → pull / sync → undo
Starting a repo
git initTurn any folder into a git repo. Run once at project root.
git clone <url>Copy a remote repo locally. Creates a new folder with everything inside.
git clone <url> .Clone into current directory instead of a subfolder. Note the dot.
git remote add origin <url>After git init, link your local repo to a GitHub remote. Do this once.
git remote -vConfirm which remote URL your repo points to. First thing to check when push fails.
Branching
git branchList all local branches. Star = current branch.
git branch -aList local and remote branches.
git checkout -b <name>Create and switch to a new branch in one move. Always branch before changing anything.habit
git checkout <name>Switch to an existing branch.
git branch -d <name>Delete a merged branch locally. Use -D to force-delete unmerged.
Staging & committing
git statusSee what's changed, staged, or untracked. Run this constantly.
git diffSee exact line-level changes before staging. Use git diff --staged after adding.
git add -pStage changes hunk by hunk interactively. Never blindly add everything.best practice
git add <file>Stage one specific file.
git add .Stage everything. Fast but sloppy — use only when you're certain.careful
git commit -m "<message>"Commit staged changes. Be specific: what + why, not "update."
git commit --amendEdit the last commit message or add forgotten files. Before pushing only.pre-push only
Pushing to GitHub
git push origin <branch>Push your branch to GitHub. Required on first push of any new branch.
git push -u origin <branch>Push and set upstream tracking in one shot. Future pushes just need git push.use this
git pushPush current branch once tracking is set (after the -u push above).
git push --force-with-leaseSafer force push — only overwrites if no one else pushed since you pulled.use carefully
Pulling & syncing
git pullFetch + merge latest from remote. Run before starting any new work.
git pull --rebaseFetch + rebase instead of merge. Cleaner history, fewer merge commits.cleaner
git fetchDownload remote changes without merging. Safe way to see what's out there.
git merge <branch>Merge a branch into your current one. Usually done via PR on GitHub instead.
Inspecting history
git log --onelineClean one-line commit history. Add -10 to limit to last 10.
git log --oneline --graphShows branch and merge structure visually in the terminal.
git show <commit-hash>See exactly what a specific commit changed.
git diff HEAD~1Compare working tree against the previous commit, not just uncommitted changes.
git tag v1.0Mark a release point. Useful for tagging validated builds in a regulated workflow.
git blame <file>See who last changed each line of a file. Useful for hunting a bug.
Undoing things
git restore <file>Discard unstaged changes to a file. Permanent — reverts to last commit state.
git restore --staged <file>Unstage a file without losing the changes.
git stashShelve all uncommitted changes temporarily. Cleans working directory without losing work.
git stash popBring stashed changes back.
git stash listSee all your stashes. You will forget what's stashed — check here.
git revert <commit-hash>Create a new commit that undoes a previous one. History stays intact.safe undo
git reset --soft HEAD~1Undo last commit but keep changes staged. Safe.
git reset --hard HEAD~1Wipe last commit and all changes. No recovery — except reflog.destructive
git reflogThe safety net. Shows every HEAD move — recover commits even after a hard reset.lifesaver
git cherry-pick <commit-hash>Pull one specific commit onto your current branch without merging the whole thing.
GitHub auth & config
git config --global user.name "name"Set your identity once. Shows on every commit.
git config --global user.email "email"Must match your GitHub account email.
git config --listSee all current git config values. Debug auth issues here first.
gh auth loginGitHub CLI — authenticate with PAT or browser. Easiest auth method.gh cli
gh repo createCreate a new GitHub repo from the terminal without touching the browser.gh cli
Safety & setup
.gitignoreA file (not a command). Lists what git should never track — .env, secrets, PAT tokens, PHI samples, build junk. In a regulated context this is your first line of defense against committing credentials.critical
git rm --cached <file>Stop tracking a file you committed by accident, without deleting it locally. Then add it to .gitignore.
git config --global alias.co checkoutMake your own shortcuts. After this, git co works like git checkout.
git tag --listList all tags in the repo.
Navigation & files
pwdPrint working directory — where am I right now.
ls -laList all files including hidden ones (like .env). Shows permissions and sizes.
cd <path>Change directory. cd .. goes up one level. cd - goes back to last directory.
mkdir -p <path>Create a directory (and any missing parent directories). The -p prevents errors if it exists.
cp -r <src> <dest>Copy a file or folder recursively.
mv <src> <dest>Move or rename a file or folder.
rm -rf <path>Delete a file or folder recursively and forcibly. No recycle bin — permanent.destructive
cat <file>Print a file's contents to the terminal.
head -n 20 <file>Show first 20 lines of a file.
tail -f <file>Stream a file live as it grows. Essential for watching logs.use this
Process & system
ps aux | grep <name>Find a running process by name.
kill -9 <pid>Force-kill a process by its ID. Get the PID from ps aux or lsof.
lsof -i :3000Find what process is squatting on port 3000 (or any port). Replace with your port.use this
ctrl+cInterrupt (stop) a running process.
ctrl+zSuspend a running process. Resume with fg.
Search & piping
grep -r "text" .Search for text across all files recursively from current directory.
grep -r "text" . --include="*.py"Narrow grep to a specific file type.
cat <file> | grep "error"Pipe file output into grep to filter. Chain with | tail -20 to limit results.
| wc -lCount lines from piped output. Append to any command. e.g. grep -r "TODO" . | wc -l
find . -name "*.json"Find all files matching a pattern recursively.
diff <file1> <file2>Compare two files line by line, outside of git.
du -sh <dir>How big is this folder. The -sh gives a single human-readable total.
df -hDisk space remaining across mounted drives, human-readable.
Shell shortcuts
ctrl+rReverse-search your command history interactively. Start typing, it finds the match. Huge time saver.use this
!!Rerun the last command. Pairs perfectly with sudo !! when you forgot sudo.
history | grep <term>Search everything you've ever typed for a command you can't remember.
chmod +x <file>Make a script executable. You'll hit this every time you write a shell script.
watch <command>Rerun a command every 2 seconds and watch the output change live.
<cmd> | xargs <cmd2>Feed the output of one command as arguments to another.
Environment & packages
echo $PATHSee your current executable search path. Where the shell looks for commands.
export VAR=valueSet an environment variable for the current session.
which pythonConfirm which python (or node, pip, etc.) the shell is actually using. Check this when package installs seem to disappear.
pip install <pkg> --break-system-packagesInstall Python packages without virtualenv conflicts. Always include this flag.
pip list | grep <name>Confirm a package actually installed.
npm installInstall all dependencies from package.json.
npm run devStart development server (or whatever script is defined in package.json).
Claude Code CLI
claudeStart an interactive Claude Code session in the current directory. Context is set by your project files.
claude -p "<prompt>"One-shot prompt — runs and exits without an interactive session. Good for scripted tasks.non-interactive
claude --resumePick up the last session where you left off.
claude --model <model>Specify which model to use for the session.
/helpIn-session command. Lists available slash commands.
/clearClear conversation history without ending the session.
/compactSummarize and compress context when the window gets long.
Setup & config
npm install -g @anthropic-ai/claude-codeInstall Claude Code globally. Requires Node.js.
which claudeConfirm the binary is installed and on your PATH.
cat AGENTS.mdRead the project-level agent instructions. This is how you configure sub-agent behavior and skill routing.
cat CLAUDE.mdProject-specific Claude instructions. Equivalent to a persistent system prompt for the repo.
Useful patterns
claude -p "summarize all TODOs in this repo"One-shot repo scan. No back and forth.
claude -p "read /docs/rubric.md and flag logic gaps"Point Claude at a specific file with a specific task.
claude -p "write tests for scoring.py"Generate tests for a file. Review before running.
claude --resume && git add -pResume a session then manually review and stage what Claude changed.best practice
// Scenario 1 — Start a new project and push to GitHub
Create repo locally
# Create folder and init mkdir signal-detection && cd signal-detection git init # Create GitHub repo via CLI (no browser) gh repo create signal-detection --private --source=. --remote=origin
First commit and push
git checkout -b main git add -p # review every change before staging git commit -m "initial: add fraud signal pipeline scaffold" git push -u origin main # sets tracking — future pushes just need git push
// Scenario 2 — Feature branch workflow
Branch, build, push
git pull # sync before touching anything git checkout -b feature/cpu-anomaly-signal # ... make changes ... git status git diff git add -p git commit -m "add cpu anomaly z-score signal to fraud pipeline" git push -u origin feature/cpu-anomaly-signal
After PR merged, clean up
git checkout main git pull git branch -d feature/cpu-anomaly-signal # delete local branch
// Scenario 3 — Something broke, find the error
Watch logs live
tail -f app.log # or stream and filter at the same time: tail -f app.log | grep "ERROR"
Hunt down the source
grep -r "KeyError" . --include="*.py" # which file throws it lsof -i :8080 # port squatting on dev server kill -9 <pid> # evict it
// Scenario 4 — Claude Code session on a project
Start and do work
cd ~/projects/patient-orchestrator claude # inside session: # "read the scoring rubric in /docs and flag logic gaps" # "write tests for risk_scoring.py"
Review and commit Claude's changes
git diff # see what Claude changed git add -p # stage only what looks right git commit -m "refactor: tighten refill risk scoring thresholds" git push
// Scenario 5 — Undo something
Oops, wrong commit (not pushed)
git log --oneline -5 # find the bad commit hash git reset --soft HEAD~1 # undo commit, keep changes staged # or nuclear if you want everything gone: git reset --hard HEAD~1
Oops, already pushed (shared branch)
git revert <commit-hash> # creates a new undo commit — safe git push # history stays clean for collaborators
// Scenario 6 — You committed a secret by accident
Not pushed yet
# Stop tracking the file, keep it locally git rm --cached .env echo ".env" >> .gitignore git commit -m "remove .env from tracking, add to gitignore"
Already pushed
# The secret is now in history — rotate it immediately. # Removing it from git does NOT make it safe once pushed. # 1. Revoke/rotate the credential at the source (PAT, API key) # 2. Then clean history (BFG or git-filter-repo) — loop in your team
